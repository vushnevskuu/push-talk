# Analysis Playbook

## Core Principle

Treat the vault as evidence, not as a place to improvise. Separate:

- Fact: what is explicitly present in notes, tasks, tags, frontmatter, or links.
- Interpretation: what the pattern likely means.
- Recommendation: what to do next.

## Review Lenses

Use these lenses during every substantial review:

- Idea: a note or cluster that contains a recoverable thesis, hypothesis, or concept worth revisiting.
- Project: a note or cluster with an execution goal, deliverable, deadline, or task structure.
- Theme: a repeated topic, tension, interest, or question across multiple notes.
- Cleanup candidate: a stub, duplicate, abandoned fragment, or low-signal note that needs merge, rewrite, archive, or deletion review.

## Suggested Review Flow

1. Run the helper script to get a current snapshot.
2. Read the highest-signal notes behind the snapshot.
3. Mark which entities are ideas, projects, themes, and cleanup candidates.
4. Estimate current status, confidence, and last meaningful movement.
5. Decide whether the user needs review, planning, cleanup, or a targeted follow-up.
6. Write the outcome into `Memory.md`.

## Idea Evaluation

Call an idea strong when most of these are true:

- It has a clear thesis in one or two sentences.
- It connects to existing interests, notes, or projects.
- It contains a non-obvious angle, hypothesis, or decision.
- It can be turned into a concrete next step.
- It appears more than once or keeps resurfacing.

Useful status labels:

- `inbox`
- `exploring`
- `active`
- `blocked`
- `paused`
- `dormant`
- `implemented`
- `dropped`
- `archived`
- `likely-active`
- `likely-dormant`
- `possible-implemented`

## Dormant And Forgotten Work

Surface an item as dormant when:

- it looks valuable,
- it has not moved for a meaningful period,
- it still has open questions, tasks, or unrealized leverage,
- and the vault shows repeated relevance.

When surfacing dormant work, always explain why it matters now.

## Cleanup Heuristics

Use this safety order:

1. Keep
2. Merge
3. Rewrite
4. Archive
5. Delete

Ask the user before destructive cleanup when confidence is not high.

High-risk cleanup signals:

- a very short note with no context,
- duplicate title or duplicate intent,
- no links, no tasks, and no follow-up,
- old fragment that never became a real idea.

## Statistics That Matter

Do not stop at raw numbers. Translate them into meaning.

Useful metrics:

- note count and word count
- open and completed tasks
- top themes, tags, and folders
- orphan notes and low-link notes
- dormant project and idea candidates
- duplicate titles or duplicate intent
- recent note activity
- skill XP totals and skill momentum
- workouts per week and nutrition consistency
- latest body metrics if the user tracks them

## Planning Rules

When the user asks for a plan, build it from vault evidence:

- tie the goal to notes or projects already present,
- give `3-7` concrete steps,
- include one minimal next step,
- mention what not to touch right now,
- note the main blocker or uncertainty when relevant.

## Default Response Structure

Use:

1. Situation
2. Why it matters
3. Dormant potential
4. Next actions
5. Memory updates
