# Модель распространения и «закрытие» кода

## Что реально возможно

1. **Исходники и Git**  
   Чтобы код не был публичным: сделайте репозиторий **приватным** на GitHub (или перенесите на закрытый Git). Релизы можно оставить только как **ZIP с `.app`**, без исходников.

2. **Бинарник всё равно можно изучать**  
   Любое нативное macOS-приложение можно дизассемблировать или патчить. Вы **не получите** такую же защиту, как у серверного SaaS.

3. **Публичная сборка сейчас бесплатная**  
   - Сайт: лендинг, FAQ, статический **`VoiceInsert-macos.zip`**.  
   - Опционально: ссылка на донат (`NEXT_PUBLIC_DONATION_URL` в Vercel) — только добровольная поддержка, не гейтинг функций.  
   - В **`Resources/Info.plist`** ключ **`VoiceInsertEntitlementBaseURL`** оставлен **пустым** — приложение **не** требует токена и не звонит на API лицензий.

4. **Опциональный биллинг (если вернёте позже)**  
   В репозитории остаются маршруты Next.js (Airwallex, claim, entitlement) и код в приложении: при **непустом** `VoiceInsertEntitlementBaseURL` (подстановка через `VOICEINSERT_ENTITLEMENT_BASE_URL` в `Scripts/build_app.sh`) снова включается проверка токена и `GET /api/entitlement`.

5. **Локальная разработка**  
   - Пустой **`VoiceInsertEntitlementBaseURL`** в plist (как по умолчанию), **или**  
   - **`VOICEINSERT_SKIP_ENTITLEMENT=1`** в окружении (см. `web/README.md`).

6. **Релизы GitHub Actions (теги `v*`)**  
   Workflow **не** задаёт `VOICEINSERT_ENTITLEMENT_BASE_URL`: публикуемый ZIP совпадает с бесплатной моделью. Чтобы снова вшить URL биллинга, добавьте переменную обратно в `release.yml` и пересоберите.

7. **Лицензия**  
   Файл `LICENSE` в корне — юридический текст для **кода в репозитории**. Для проприетарной модели обычно **меняют лицензию** и раздают только бинарник под своими условиями.

## Краткий чеклист

- [ ] На лендинге и в FAQ тексты совпадают с фактической моделью (сейчас: бесплатный ZIP, без токена).  
- [ ] При включении биллинга: `DATABASE_URL`, Airwallex, webhooks, совпадение URL в plist и на сайте.  
- [ ] По желанию: Developer ID + нотаризация для меньше предупреждений Gatekeeper.
