# Коммерческая модель и «закрытие» кода

## Что реально возможно

1. **Исходники и Git**  
   Чтобы код не был публичным: сделайте репозиторий **приватным** на GitHub (или перенесите на закрытый Git). Релизы можно оставить только как **ZIP с `.app`**, без исходников.

2. **Бинарник всё равно можно изучать**  
   Любое нативное macOS-приложение можно дизассемблировать или патчить. Вы **не получите** такую же защиту, как у серверного SaaS. Задача биллинга — остановить **честных** пользователей без оплаты и автоматизировать отключение после конца периода, а не победить реверс-инжиниринг.

3. **Как у вас сейчас устроена оплата**  
   - Сайт: Airwallex (старт триала **$1**, дальше **подписка**, периоды приходят из биллинга).  
   - Сервер: `GET /api/entitlement` проверяет токен и статус подписки в БД.  
   - Приложение: если в `Info.plist` задан непустой **`VoiceInsertEntitlementBaseURL`**, перед диктовкой требуется успешная проверка; токен хранится в **Keychain**.

4. **Локальная разработка без подписки**  
   - Оставьте **`VoiceInsertEntitlementBaseURL`** пустым в `Resources/Info.plist`, **или**  
   - Запускайте с **`VOICEINSERT_SKIP_ENTITLEMENT=1`** в окружении (см. `web/README.md`).

5. **Релизный ZIP с обязательной подпиской**  
   В GitHub Actions для тегов `v*` в `release.yml` задано `VOICEINSERT_ENTITLEMENT_BASE_URL=https://push-talk.vercel.app`.  
   Скрипт `Scripts/build_app.sh` подставляет это значение в собранный `Info.plist`.  
   Смените URL, если продакшен-домен другой.

6. **Лицензия**  
   Файл `LICENSE` в корне (если остался MIT) — юридический текст для **кода в репозитории**. Для закрытой коммерческой модели обычно **меняют лицензию** на проприетарную и раздают только бинарник под своими условиями. Имеет смысл пройтись с юристом, если важны формулировки для B2C.

## Краткий чеклист

- [ ] Репозиторий приватный (или только CI с доступом).  
- [ ] В Vercel настроены Airwallex, webhooks, `DATABASE_URL`, публичный URL совпадает с `VOICEINSERT_ENTITLEMENT_BASE_URL` в релизной сборке.  
- [ ] На лендинге и в FAQ тексты совпадают с фактической моделью (подписка, токен, блокировка без плана).  
- [ ] По желанию: Developer ID + нотаризация для меньше предупреждений Gatekeeper.
